# Personal Memory — Personal Computer Assistant

Your persistent personal knowledge base and continuity layer.

## Core Functions
- **Distill**: At end of meaningful sessions or on command, use distill patterns (or ormus-absorb equivalent) to extract learnings, decisions, threads into structured memory (files in ~/.grok/memory or personal projects, AGENTS.md extensions).
- **Recall**: Before briefs/tasks, pull relevant context (projects, relationships, ongoing intentions from AGENTS, previous distills).
- **Update**: Proactively suggest or apply updates to personal rules, rhythms (14-rhythm), polarity balances, etc.
- **Query**: Answer "what's my current state on Y?" from memory.

## Integration
- Works with daily-brief (pull context), task-orchestrator (carry threads), comms-assistant (log interactions).
- Ties to reality-os AGENTS.md as the living personal doctrine.
- Use sessions, background for long-term accumulation.
- ormus personal projects as external memory (e.g., ormus-checkin for habits).

## Output
Always end memory operations with a concise updated summary + suggested user review if high-stakes.

## Rules
- Only user's sovereign data.
- Structured, searchable (markdown, perhaps simple DB via MCP).
- Periodic clean (calcinate style on memory bloat).
- Hermetic: memory serves the living I, not mechanical accumulation.

Call at session start/end or explicitly.
